# 00. IRIS-workflow 概要

## 目的

IRIS-workflow は、**日常の様々な動作をワークフローにできる** パーソナル・オートメーション基盤です。
ユーザは「何かのトリガが起きたら、AI を呼び出して処理し、デバイスのウィジェットや動きに反映する」というフローを、コードを書かずに組み立てられます。

## ひと目で分かる全体像

```
┌──────────────┐     ┌────────────────────┐     ┌──────────────────────┐
│  入力         │ →  │  AI-centric         │ →  │ デバイスの動作・表示   │
│  Interface   │    │  Workflow (DAG)     │    │ Widget / IoT / 通知   │
└──────────────┘     └────────────────────┘     └──────────────────────┘
   時刻 / 音声           Claude Code を            iPhone / Win / IoT
   Webhook / 手動         中央AIに据えた             ホーム画面 / トレイ
   センサ / ファイル変更    DAG エンジン              サウンド / 機器制御
```

## 3つのレイヤ

| レイヤ | 内容 | 実装 |
| --- | --- | --- |
| **ホスト** | ユーザのメインPCで常駐するワークフローエンジン | Rust + Axum + PostgreSQL |
| **デバイス** | 入力源 / 出力先となるデバイス群 | iOS / Windows / IoT |
| **AI** | ワークフローの中で頭脳として呼ばれる | Claude Code (CLI) を subprocess で駆動 |

## 主な概念 (この docs で登場するキーワード)

- **Host-PC モデル** — クラウドではなくユーザPC上に常駐する Rust デーモン
- **VPN メッシュ** — Tailscale などで全デバイスを同一ネットワーク化
- **Workflow DSL** — DAG を YAML で記述
- **Server-driven UI (SDUI)** — UI スキーマをサーバが配信
- **Permission MCP** — Claude Code の権限要求をデバイスへプッシュ通知して承認
