# 01. ビジョンとユースケース

## 目的

IRIS-workflow が **何を解決し、誰のために存在するのか** を定義します。

## ビジョン

> 自分専属の AI アシスタントが、自分のデバイス群を統合的に動かす世界を、自分のPCから自分で組み立てられるようにする。

クラウドSaaSや既存の自動化サービス (IFTTT, Zapier, n8n) には次の課題があります:

- AI を使うとき、API キー課金が前提でローカル開発との接続が弱い
- データが第三者クラウドを通過するためプライバシ・機密性に懸念
- ウィジェットや IoT との統合は限定的
- 「自分の Claude Code 環境」をそのまま活かせない

IRIS-workflow は、**自分のPCを中央集権 (=ホスト) に据え、Claude Code を頭脳に据える** ことで、上記の課題を裏返しに解決します。

## ターゲットユーザ

- 個人開発者・パワーユーザ
- Claude Code を日常的に使っているエンジニア
- 自宅に IoT デバイスを揃えているハードウェア愛好家
- 「自分のための AI 統合環境」を作りたい人

## ユースケース例

### Case 1: 朝の準備自動化

- **トリガ**: 平日 6:30
- **ワークフロー**:
  1. Claude Code がカレンダー API から今日の予定を取得
  2. 天気 API + 服装提案を生成
  3. Spotify でルーティン用プレイリストを再生
- **出力**:
  - iPhone のロックスクリーンウィジェットに「今日のサマリ」を表示
  - Windows のタスクトレイに同内容を表示
  - スマート照明をゆっくり点灯

### Case 2: 会議サマリ → iPhone ウィジェット

- **トリガ**: ZoomやTeams会議が終了した（Webhook）
- **ワークフロー**:
  1. 議事録ファイルを Claude Code に渡す
  2. 要約・次のアクション・カレンダー追加候補を生成
  3. SDUI で「次アクション一覧」ウィジェット仕様を作成
- **出力**:
  - iPhone ホーム画面のウィジェットが更新される
  - 完了アクションは Slack に投稿

### Case 3: IoT 照明連動 (深夜執筆モード)

- **トリガ**: VSCode/Cursor を 23時以降に起動
- **ワークフロー**:
  1. Claude Code に「執筆モード」プリセットを問い合わせ
  2. 照明・BGM・通知抑止のコマンドを発行
- **出力**:
  - スマート照明を暖色 30%
  - HomePod で集中BGM
  - Windows の通知を「サイレント」に
  - iPhone を DND モードに

### Case 4: コードレビュー後のフォロー

- **トリガ**: GitHub PR にコメントが付いた (Webhook)
- **ワークフロー**:
  1. PR + コメントを Claude Code に渡し、対応案を生成
  2. 簡単な修正なら自動で PR を更新
- **出力**:
  - 重要コメントだけ iPhone に通知
  - 自動修正したものは「対応済み」を Slack 通知

## 非ゴール (現時点)

- マルチテナント (組織向け SaaS) としての提供
- ノーコード初心者向けの完全グラフィカル UI (まずは YAML + 簡易GUI)
- モバイル/Mac の他ホスト対応 (まずは Windows ホストにフォーカス)

## 次に詰めること

- カレンダー・天気・Slack 等の外部 API 抽象化レイヤ
- ユーザが既に持っている IoT (家庭環境) との接続性調査
